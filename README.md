# 微信公众号文章获取工具

一键获取任意微信公众号的全部文章 URL 及纯文本正文。
本工具提供两项核心功能：
1. **单篇文章读取**：提供任意单篇微信公开文章的链接，即可稳定解析出标题与正文纯文本。
2. **公众号全量提取**：通过微信公众平台配合本地登录凭证，可一键提取目标公众号的全部（或指定时间段）历史文章及纯文本正文内容。

**本项目已深度针对 AI 助理（Agent）进行优化，强烈建议你无需自己敲代码，直接用自然语言命令你的智能 Agent 帮你完成所有操作！**

## 快速开始使用流程

### 1. 准备环境
你可以直接对 Agent 说（示例）：
> **“帮我检查并安装这个项目的依赖（requirements.txt 和 playwright 内核）”**

或者在终端输入：
```bash
pip install -r requirements.txt
playwright install chromium
```

### 2. 告诉 Agent 你想抓谁
你需要明确下达目标，不要自己再去改配置文件了，直接对 Agent 说（示例）：
> **“请帮我抓取公众号《数字生命卡兹克》最新的 10 篇文章。”**
> 或者：“帮我获取《某某科技》2026-03-01 以来的所有文章全文。”

或者在终端输入：
```bash
# 获取默认数量的文章
python crawler.py --nickname "数字生命卡兹克"

# 仅获取最新的 10 篇文章
python crawler.py --nickname "数字生命卡兹克" --max 10

# 获取从指定日期以来的所有文章
python crawler.py --nickname "数字生命卡兹克" --since 2026-03-01

# 日期 + 数量组合使用
python crawler.py --nickname "数字生命卡兹克" --since 2026-03-01 --max 50
```

### 3. 获取登录凭证 (核心步骤)
由于微信公众号文章对全量获取有严格的扫码与 IP 安全限制。当 Agent 准备帮你启动提取程序前，**你必须要明确告诉 Agent 你当前的机器环境**：

#### 场景 A：你在自己的个人电脑（Windows/Mac）上跑
直接对 Agent 下发指令（示例）：
> **“我是在本地运行，请帮我启动全自动浏览器扫码获取程序。”**

Agent 接收到指令后，会自动帮你执行脚本。此时你的电脑上会弹出一个 Chrome 浏览器网页，你只需要**拿出手机微信扫一扫屏幕上的二维码**即可，Agent 会全自动帮你完成剩下的目录翻页和正文批量下载。

或者在终端输入：
```bash
python crawler.py --nickname "目标公众号"
```
（随后按回车键触发 Playwright 自动扫码流程）

#### 场景 B：你在云服务器（无屏幕的 Linux / 云机房节点）上跑
微信对云厂商 IP 节点及无头浏览器风控极其严格，切勿在云端直接执行登录！因此你必须**告诉 Agent 你在云端，并亲自在本地准备好凭证**：
> **“我是在云服务器跑，我来为你提供凭证，请准备好接收参数。”**

**接下来你要做的手动操作：**
1. 用你自己本地个人电脑的浏览器登录 [微信公众平台](https://mp.weixin.qq.com)。
2. 按 `F12` 打开开发者工具，切到 **Network (网络)** 标签页。
3. 刷新页面，随便点击左侧一个以 `cgi-bin/` 开头的网络请求。
4. 从 URL 地址里复制你要的 `token=数字`。
5. 往下看，在 **Request Headers (请求标头)** 里找到 `Cookie: ` 字段，把后面一整串极长字符完整复制下来。
6. 直接把这两个文本扔给 Agent（示例）：
> **“这是我的 Cookie：xxxxx，这是我的 Token：123456。凭证给你了，直接帮我跑静默抓取吧！”**

Agent 会极度聪明地帮你将这些长文本拼接成底层脚本需要的复杂 JSON 数据结构并在后台安静执行！

或者在终端输入（将参数替换为你的实际值）：
```bash
python crawler.py --nickname "目标公众号" --credentials '{"cookie":"填写刚才你的长串Cookie","token":"填写刚才你的Token"}'
```

### 4. 自动产生全量结果
程序在 Agent 的接管下已实现全自动连贯执行：提取完文章基础目录后，**会自动无缝进入第二阶段**，全自动将列表里所有文章的纯文本详情（Content）逐篇下载。

最终，包含链接和全文的正果会保存在 `output/` 目录下，文件名为 `xxx_full_时间戳.json`，包含了标题、URL、发布时间以及完整的正文。

## ⚠️ 核心使用受限说明与免责注意事项
为了能长久稳定配合 Agent 使用本脚本，请您务必仔细阅读以下机制红线：

1. **凭证不具备持久性（几小时即报废）**：
   无论你怎么提取 `Cookie` 和 `Token`，它都**绝不是永久的**！微信的安全机制会使扫码登录态在 **几小时甚至几十分钟（如果你剧烈变更了IP）** 内自动过期。一旦 Agent 跟你抱怨报错需要重新提取，说明凭证已经自然衰败，你除了立马跑去公众号后台重新扫一次，没有任何奇技淫巧能长期绕过。**世界上没有永久免扫码拿数据的方法**。
2. **云服务器自动化受限红线（封号风险）**：
   绝不允许让 Agent 在阿里云、腾讯云机房等非常理 IP 地址上擅作主张跑无头浏览器去自动化登录。一旦触发云 IP 严控规则，轻则无限弹验证码，重则该公众号可能直接被封禁网页端操作权限。请绝对老实地遵循【场景 B】在本地偷好 Cookie 喂给它跑！
3. **彻底获取不到阅读量与点赞等社群互动数据**：
   受限于只能抓网页后台的机制底线，本程序联合 Agent **只能完美获得绝对的文章元数据列表和文章内部纯文本（Content）**。对于一篇文的阅读数、点赞量、或者评论留言，全盘是被微信封死在手机或微信 PC 版客户端的私有下发协议里的，此类纯外部网页提取工具不管怎么试都是瞎折腾。
4. **过于暴力的请求频率将遭制裁**：
   单脚本串行请求间隔默认是 **3 秒**，若你需要一口气抽取长达几千篇的骨灰级老号文章大全，疯狂发收极易导致账号 IP 被微信官方拦截 24 小时。遇到此类重度任务，请吩咐 Agent 去帮你主动调高代码里或 config 里的并发等待间隔参数。
5. 本仓库及附属 Agent 服务仅供个人非商用的学习编码与研究交流分析，请时刻敬畏数据所有权与原作者著作版权。
